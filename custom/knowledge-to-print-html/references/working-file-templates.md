# Working File Templates

Use these lightweight templates when the working files start drifting or when you need a clean starting shape quickly.

## `raw-input.md`

Purpose: preserve the original sample exactly before analysis.

```md
# Raw Input

## Source
- user message, pasted notes, uploaded draft, or mixed material

## Preservation Rule
Keep the original order and wording. Do not clean up, summarize, or regroup this sample here.

## Untouched Sample
[paste the original text verbatim]
```

## `brief.md`

Purpose: normalize the task after `raw-input.md` is safely preserved.

```md
# Brief

- topic slug:
- audience:
- reading goal:
- research mode: comprehensive | constrained
- constrained reason:

## Core Knowledge Point List
- point 1
- point 2

## Draft Prose Worth Preserving
- excerpt or note

## Gaps / Checks / Visual Opportunities
- missing explanation
- factual check
- diagram opportunity
```

## `research.md`

Purpose: keep research organized by core knowledge point.

```md
# Research

## Core Knowledge Point: [name]
- authoritative explanation:
- boundaries / misconceptions:
- example / application / counterexample:
- sources:
- retrieval date:
```
